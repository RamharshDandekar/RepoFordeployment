What is Collision?

A collision occurs when two different keys produce the same hash value and are assigned to the same slot in a hash table. Since this slot is already occupied, a technique is needed to handle this situation.

Collisions Example

Collisions Resolution Methods

There are two main types of methods to resolve collisions:

  1. Separate Chaining: Each slot of the hash table contains a linked list where all colliding keys are stored.
  2. Open Addressing: If a collision occurs, the algorithm tries to find another open slot in the table to store the key.

What is Separate Chaining?

Separate chaining is a collision resolution technique where each slot in the hash table contains a linked list. When a collision occurs, the new key-value pair is simply added to the linked list at that slot, allowing multiple values to be stored at the same index.
