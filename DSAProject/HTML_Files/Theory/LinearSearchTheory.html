Linear Search

Introduction to Linear Search

Linear Search, also known as Sequential Search, is a simple search algorithm that checks each element in a list sequentially until the desired element is found or the list ends. It is easy to implement and understand.

Problem Statement

Given an array of integers and a target value, the goal is to determine if the target exists in the array and return its index. If the target does not exist, return -1.

How Linear Search Works

  • Start from the first element in the array.
  • Compare the current element with the target value.
  • If they match, return the index of the current element.
  • If they do not match, move to the next element and repeat until the target is found or the end of the array is reached.
  • If the end is reached without finding the target, return -1.

Example

For the array [3, 5, 2, 8, 6] and target value 8, Linear Search will return the index 3, since 8 is found at that position.

Complexity Analysis

The time complexity of Linear Search is O(n), where n is the number of elements in the array. The space complexity is O(1), as it uses a constant amount of space.

Applications of Linear Search

  • Searching for an item in an unsorted list.
  • Basic checks and validations in data processing.
  • When the data set is small or the cost of sorting the data is not justified.

Variations of Linear Search

  • Finding the first occurrence of a target value.
  • Finding the last occurrence of a target value.
  • Counting the number of occurrences of a target value.

Conclusion

Linear Search is a fundamental algorithm suitable for simple search tasks. Despite its inefficiency for large datasets compared to other search algorithms, its straightforward nature makes it an excellent choice for unsorted or small lists. Understanding Linear Search lays the foundation for more complex searching algorithms.
