Hash Tables

Hash Tables Concept and Algorithm

Need of Hashing

  • It can be used to compare large amounts of data. We create the hashes for the data, store the hashes, and later compare them.
  • Hashes can be used for indexing data, especially in hash tables, to quickly locate records using the hash as a pointer.
  • Used in cryptographic applications like digital signatures.
  • Hashing can be used to generate seemingly random strings.

Hashing Basics

  • Hashing maps large data sets to a smaller table using a hashing function.
  • It converts a range of key values into a range of indexes for an array.
  • Hashing allows data updates and retrieval in constant time O(1).

Real World Examples

In these examples, elements are hashed to unique numbers, similar to keys:

  • Universities assign unique roll numbers to students, used to retrieve their information.
  • Libraries assign unique numbers to books for identifying their positions and borrowing information.
  • A pin code helps identify a region, sub-region, and so on.

Hashing Functions

A hash function maps a data set of an arbitrary size to a fixed-size data set. The values returned are called hash values or simply hashes. These hashes are mapped to hash table arrays.

    hash = hashfunc(key)
    index = hash % array_size

For example, a hash function can map strings to the sum of their ASCII characters or numbers to themselves.

Linear Probing Concept and Algorithm

What is Probing?

When two keys result in the same hash value, it causes a collision in the hash table. Probing is a method of resolving such collisions. In probing, we systematically look for the next available slot for storing the key-value pair.

Types of Probing

There are two main types of probing:

  • a. Linear Probing

    In this technique, we linearly probe through the hash table to find the next free slot after a collision.

  • b. Quadratic Probing

    Here, the distance between probes increases quadratically (i.e., we jump to increasingly distant slots to find a free space).

Quadratic Probing Concept and Algorithm

What is Quadratic Probing?

Quadratic probing is an open addressing scheme which operates by taking the original hash index and adding successive values of an arbitrary quadratic polynomial until an open slot is found. This reduces clustering issues compared to linear probing, as it avoids consecutive probing positions.
